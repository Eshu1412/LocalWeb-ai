1. Project Vision & Executive Summary

LocalWeb AI is a fully automated, multi-agent SaaS platform that identifies local businesses without an online presence, autonomously reaches out to them via AI voice calls and WhatsApp, demonstrates a ready-made sample website tailored to their business, and closes the deal — all without human intervention. Once a business agrees and pays, the site is instantly deployed and made publicly accessible.

1.1 Core Problem Being Solved
Millions of small and local businesses — general stores, restaurants, salons, clinics, repair shops — operate without any web presence. They lose customers daily to competitors who are discoverable online. Traditionally, getting a website requires hiring developers, navigating complex tools, or paying agencies thousands of dollars. Most small businesses simply never do it.

11. Compliance, Legal & Ethics

11.1 Calling & Messaging Compliance
TCPA (USA): Only call during 8am–9pm local time. Maintain a Do-Not-Call list. Get consent before automated calls.
GDPR (EU): Lawful basis for processing business contact data. Provide opt-out mechanism. Delete data on request.
WhatsApp Policy: Only use approved message templates for first contact. Never spam. Respect opt-outs immediately.
CAN-SPAM: All email outreach must have valid sender info and one-click unsubscribe.
FTC Telemarketing Rules: AI calls must disclose they are automated within the first few seconds.

11.2 Technical Safeguards
Rate limiting: Max 50 discovery calls/day, max 200 WhatsApp messages/hour per sending number
Do-Not-Contact list: Redis set of opted-out phone numbers — checked before every outreach
Call time gating: Agent checks local time zone before placing any call
Human escalation: Any call that goes beyond standard script escalates to human review queue
Data encryption: All PII encrypted at rest (AES-256) and in transit (TLS 1.3)

13. Server-Side Management Guide

This section is the operational handbook for the server-side / DevOps team. It covers every management task — from first-time project initialization and API key setup, to ongoing monitoring, secrets rotation, database administration, agent control, and emergency rollback procedures. All commands assume the project root is active and the Python virtual environment is loaded.

13.1 First-Time Deployment CLI Wizard
Run the interactive setup wizard once on a fresh server. It validates all prerequisites, collects every API key and credential, writes a .env file, initialises the database, seeds default data, and performs a smoke test before handing off to the running stack. Never hard-code secrets — the wizard stores them in Vault / Secrets Manager automatically.

13.2 API Key Configuration CLI
Use the key-manager CLI to add, rotate, validate, or remove any API key without touching .env files manually. All values are encrypted at rest and injected at runtime as environment variables. Supports AWS Secrets Manager, HashiCorp Vault, and a local encrypted .env.vault file for development.

13.3 Required API Keys Reference
All keys below must be configured before the platform can run in production. The wizard will prompt for each one in order.

13.4 Service Health & Status CLI
Use the health CLI to check the status of every microservice, agent worker, and external API integration at any time. Returns a colour-coded table with latency, error rate, and last-seen timestamps.

13.5 Database Management CLI
All database administration tasks — migrations, backups, restores, data audits, purges, and index maintenance — are performed through the db-manager CLI. Never run raw SQL against production without first taking a snapshot via this tool.

13.6 Agent Control CLI
Start, stop, pause, scale, and configure individual AI agents at runtime without restarting the entire stack. This is the primary tool for traffic shaping and incident response.

13.7 Log Management CLI
All agent activity, API call logs, errors, and pipeline events are written to structured JSON logs and streamed to the central log aggregator (Datadog / ELK). The log CLI provides fast local tailing, filtering, and export without leaving the server.

13.8 Do-Not-Contact (DNC) List Management
The DNC list is a Redis set of opted-out phone numbers checked by every outreach agent before making contact. It is also persisted in PostgreSQL for auditability. Compliance law in many jurisdictions (TCPA, GDPR) requires that opt-outs are honoured immediately and permanently.

13.9 Secrets Rotation & Security Maintenance
Rotate API keys and credentials on a schedule (recommended: every 90 days) or immediately after a suspected security incident. The rotation CLI updates Vault / Secrets Manager, redeploys affected services, and logs the rotation event for the audit trail.

13.10 Environment & Feature Flag Management
The env-config CLI manages non-secret runtime configuration: feature flags, calling windows, rate limits, outreach cooldown periods, and pipeline behaviour toggles. Changes take effect in under 10 seconds without a service restart via Redis-backed config hot-reload.

13.11 Admin User & Access Management
Create, modify, and revoke admin dashboard accounts. Supports role-based access control (RBAC) with three built-in roles: super_admin (full access), ops_manager (manage agents, view all data), and viewer (read-only dashboard access).

13.12 Deployment, Updates & Rollback
Use the deploy CLI to push new versions, perform rolling updates with zero downtime, and roll back instantly if something goes wrong. All deployments are recorded with git SHA, timestamp, and deployer identity for the audit trail.

13.13 System Diagnostics & Performance Profiling
The diagnostics CLI collects a point-in-time snapshot of the entire system state: CPU/memory, queue depths, slow queries, error rates, and API latency. Run it first whenever investigating an incident.

13.14 Stripe & Billing Management CLI
Manage Stripe products, prices, subscriptions, and webhooks from the command line. Useful for seeding plans on a new server, adjusting pricing without touching the Stripe dashboard, and reconciling payment records against the local database.

13.15 Client Site & Domain Management CLI
Manage deployed client sites, custom domains, and Vercel/Cloudflare configurations. Use when a client site has a deployment issue, needs a domain change, or requires a manual redeploy.

import asyncio, redis.asyncio as redis, json, logging	
from abc import ABC, abstractmethod	
from sqlalchemy.ext.asyncio import AsyncSession	
from db.models import BusinessLead, PipelineEvent	
	
class BaseAgent(ABC):	
    def __init__(self, redis_client, db_session):	
        self.redis = redis_client	
        self.db = db_session	
        self.logger = logging.getLogger(self.__class__.__name__)	
	
    @abstractmethod	
    async def process(self, lead_id: str, payload: dict) -> dict:	
        pass	
	
    async def emit_event(self, stream: str, lead_id: str, data: dict):	
        await self.redis.xadd(stream, {'lead_id': lead_id, 'data': json.dumps(data)})	
	
    async def update_lead_status(self, lead_id: str, status: str, notes: str = ''):	
        # Updates PostgreSQL business_leads table	
        async with self.db() as session:	
            lead = await session.get(BusinessLead, lead_id)	
            lead.status = status; lead.notes = notes	
            await session.commit()

class DiscoveryAgent(BaseAgent):	
    async def process(self, area: str, category: str) -> list[dict]:	
        # 1. Query Google Places API	
        url = 'https://maps.googleapis.com/maps/api/place/textsearch/json'	
        params = {'query': f'{category} in {area}', 'key': GOOGLE_API_KEY}	
        response = await httpx.get(url, params=params)	
        places = response.json()['results']	
	
        # 2. Enrich with phone, address, category, place_id	
        leads = []	
        for place in places:	
            lead = {	
                'name': place['name'],	
                'phone': place.get('formatted_phone_number'),	
                'address': place.get('formatted_address'),	
                'category': category,	
                'place_id': place['place_id'],	
                'lat': place['geometry']['location']['lat'],	
                'lng': place['geometry']['location']['lng'],	
            }	
            leads.append(lead)	
            await self.save_lead(lead)  # Save to PostgreSQL	
	
        # 3. Emit to verification stream	
        for lead in leads:	
            await self.emit_event('stream:verify', lead['id'], lead)	
        return leads

agents/verification_agent.py — Core Logic	
import socket, httpx	
class VerificationAgent(BaseAgent):	
    async def process(self, lead_id: str, payload: dict) -> dict:	
        name = payload['name']	
        # 1. DNS check — try common domain patterns	
        domains_to_try = [	
            f"{name.lower().replace(' ','-')}.com",	
            f"{name.lower().replace(' ','')}.com",	
            f"{name.lower().replace(' ','')}online.com",	
        ]	
        for domain in domains_to_try:	
            try:	
                socket.gethostbyname(domain)	
                r = await httpx.get(f'https://{domain}', timeout=5)	
                if r.status_code < 400:	
                    await self.update_lead_status(lead_id, 'HAS_WEBSITE')	
                    return {'has_website': True, 'url': f'https://{domain}'}	
            except: pass	
        # 2. Google search for business name + location	
        search = await serpapi_search(f'{name} {payload["address"]} official site')	
        if search['organic_results'] and 'site:' in search['organic_results'][0]['link']:	
            await self.update_lead_status(lead_id, 'HAS_WEBSITE')	
            return {'has_website': True}	
        # 3. No website found — proceed to sample builder	
        await self.update_lead_status(lead_id, 'NO_WEBSITE')	
        await self.emit_event('stream:sample_build', lead_id, payload)	
        return {'has_website': False}

agents/calling_agent.py — Twilio + ElevenLabs	
class CallingAgent(BaseAgent):	
    async def initiate_call(self, lead_id: str, payload: dict):	
        # 1. Generate personalized script with LLM	
        script = await llm.generate(f'''	
        Write a friendly 60-second phone pitch for a {payload['category']}	
        owner named [Owner]. Their business: {payload['name']}.	
        Mention we built them a free website preview at {payload['preview_url']}.	
        Keep it conversational, not salesy. End with asking if they can check WhatsApp.''')	
	
        # 2. Convert script to voice with ElevenLabs	
        audio_url = await elevenlabs.tts(script, voice_id='friendly_male_en')	
	
        # 3. Place call via Twilio with TwiML	
        twiml = f'<Response><Play>{audio_url}</Play><Record maxLength="120"/></Response>'	
        call = await twilio.calls.create(	
            to=payload['phone'],	
            from_=TWILIO_NUMBER,	
            twiml=twiml,	
            status_callback=f'{API_BASE}/webhooks/twilio/call-status'	
        )	
	
        # 4. Save call SID and wait for webhook callback	
        await self.update_lead_status(lead_id, 'CALL_INITIATED')	
        return {'call_sid': call.sid}	
	
    async def handle_call_response(self, call_sid: str, recording_url: str):	
        # Transcribe owner's response with Whisper	
        transcript = await whisper.transcribe(recording_url)	
        # Send to Negotiation Agent for processing	
        await self.emit_event('stream:negotiate', call_sid, {'transcript': transcript})

agents/negotiation_agent.py — LLM + RAG	
class NegotiationAgent(BaseAgent):	
    def __init__(self, *args):	
        super().__init__(*args)	
        self.vectorstore = PineconeVectorStore(index='faq-kb')  # FAQ knowledge base	
	
    async def process(self, lead_id: str, payload: dict) -> dict:	
        message = payload['message']	
        channel = payload.get('channel', 'whatsapp')  # 'whatsapp' or 'call'	
	
        # 1. Retrieve relevant FAQ answers from vector store (RAG)	
        relevant_docs = await self.vectorstore.similarity_search(message, k=3)	
        context = '\n'.join([doc.page_content for doc in relevant_docs])	
	
        # 2. Generate personalized response	
        response = await llm.chat([	
            SystemMessage('You are a friendly sales assistant for LocalWeb AI...'	
                         f'Context from FAQ:\n{context}'),	
            HumanMessage(message)	
        ])	
	
        # 3. Send response back via appropriate channel	
        lead = await self.get_lead(lead_id)	
        if channel == 'whatsapp':	
            await meta_api.send_text(to=lead.phone, body=response.content)	
        # 4. Check if lead is ready to pay after response	
        intent = await classify_intent(message)  # 'ready'/'objection'/'not_interested'	
        if intent == 'ready':	
            await self.emit_event('stream:payment', lead_id, lead.payload)	
        return {'response': response.content, 'intent': intent}

import stripe	
class PaymentAgent(BaseAgent):	
    async def create_payment_link(self, lead_id: str, payload: dict) -> str:	
        # 1. Create Stripe customer	
        customer = await stripe.Customer.create(	
            name=payload['name'],	
            phone=payload['phone'],	
            metadata={'lead_id': lead_id}	
        )	
        # 2. Create subscription + one-time setup fee	
        session = await stripe.checkout.Session.create(	
            customer=customer.id,	
            payment_method_types=['card'],	
            line_items=[	
                {'price': 'price_starter_monthly', 'quantity': 1},  # $49/mo	
                {'price': 'price_setup_fee',       'quantity': 1},  # $99 one-time	
            ],	
            mode='subscription',	
            success_url=f'{BASE_URL}/success?session_id={{CHECKOUT_SESSION_ID}}',

        )	
        # 3. Send payment link via WhatsApp

            to=payload['phone'],	
            body=f'Great! Here is your secure payment link:\n{session.url}\n'	
                 f'After payment, your website goes live within 24 hours!'	
        )	
        await self.update_lead_status(lead_id, 'PAYMENT_LINK_SENT')	
        return session.url	
	
    async def handle_payment_success(self, session_id: str):	
        # Called by Stripe webhook on successful payment	
        lead = await self.get_lead_by_stripe_session(session_id)	
        await self.update_lead_status(lead.id, 'PAID')

class BuilderAgent(BaseAgent):	
    async def process(self, lead_id: str, payload: dict) -> dict:	
        category = payload['category']	
        # 1. Clone template repository for the business category	
        repo_path = f'/tmp/build-{lead_id}'	
        shutil.copytree(f'templates/{category}', repo_path)	
	
        # 2. Inject AI-generated content into template files	
        content = await self.generate_full_content(payload)	
        await self.inject_content(repo_path, content)	
	
        # 3. Register custom domain via Cloudflare API	
        domain = f"{payload['slug']}.com"  # e.g. joes-pizza.com	
        await cloudflare.create_dns_record(domain)	
	
        # 4. Deploy to Vercel with custom domain	
        project = await vercel.create_project(	
            name=payload['slug'],	
            framework='nextjs',	
            repo_path=repo_path	
        )	
        await vercel.add_domain(project.id, domain)	
        deployment = await vercel.deploy(project.id)	
	
        # 5. Wait for deployment to go live, then notify QA Agent	
        await deployment.wait_ready()	
        await self.emit_event('stream:qa', lead_id,	
            {**payload, 'live_url': f'https://{domain}'})	
        return {'live_url': f'https://{domain}', 'project_id': project.id}

agents/qa_agent.py — Playwright + Lighthouse + LLM	
class QAAgent(BaseAgent):	
    async def process(self, lead_id: str, payload: dict) -> dict:	
        url = payload['live_url']	
        results = {}	
	
        # 1. Playwright end-to-end checks	
        async with async_playwright() as p:	
            browser = await p.chromium.launch()	
            page = await browser.new_page()	
            await page.goto(url)	
            results['title'] = await page.title()	
            results['links_ok'] = await self.check_all_links(page)	
            results['mobile_ok'] = await self.check_mobile_viewport(page)	
            await browser.close()	
	
        # 2. Lighthouse performance + SEO audit	
        lighthouse = await run_lighthouse(url)	
        results['performance'] = lighthouse['categories']['performance']['score']	
        results['seo_score'] = lighthouse['categories']['seo']['score']	
        results['accessibility'] = lighthouse['categories']['accessibility']['score']	
	
        # 3. LLM content quality review	
        page_html = await fetch_html(url)	
        review = await llm.chat(f'Review this website HTML for quality, professionalism,	
            and accuracy. Score 1-10 and flag any issues.\n{page_html[:3000]}')	
        results['llm_score'] = extract_score(review)	
        results['llm_feedback'] = review	
	
        # 4. Auto-approve if scores meet threshold, else flag for fix	
        if all([results['performance'] > 0.8, results['seo_score'] > 0.8,	
                results['llm_score'] >= 8]):	
            await self.emit_event('stream:seo', lead_id, {**payload, 'qa': results})	
            await self.update_lead_status(lead_id, 'QA_PASSED')	
        else:	
            await self.emit_event('stream:build_fix', lead_id, {**payload, 'qa': results})	
        return results
